# StashFlow Settings Guide

The Settings Hub is divided into several sections to customize your experience.

## Server Settings (`/settings/server`)
Manage your connection to the Stash server.
*   **Server URL:** Set the base URL of your Stash server.
*   **API Key:** Configure the API key for secure access.

## Playback Settings (`/settings/playback`)
Configure video playback behavior.
*   **Prefer Scene Streams:** Toggle whether to prioritize direct scene streams over transcoded ones.
*   **Autoplay Next:** Automatically play the next scene in the queue or list.
*   **Show Video Debug Info:** Display technical stream and playback diagnostics on screen.
*   **Seek Interaction Mode:** Choose between 'Double-tap to seek' and 'Drag to seek' for the video player.
*   **Background Playback:** Allow audio/video to continue playing while the app is in the background.
*   **Native Picture-in-Picture (PiP):** Enable the native OS PiP mode for the video player.
*   **Subtitle Settings:** Customize subtitle language, font size, position, and text alignment.

## Appearance Settings (`/settings/appearance`)
Customize the visual theme and colors.
*   **Theme Mode:** Switch between Light, Dark, and System Default themes.
*   **Grid Layout Settings:** Configure column counts and aspect ratios based on screen size (Mobile vs Tablet).

## Interface Settings (`/settings/interface`)
Customize navigation and layout behaviors.
*   **Scene Layout Style:** Choose the default layout for Scenes (List, Grid, or TikTok style).
*   **Visible Navigation Tabs:** Select which tabs (Scenes, Performers, Studios, Tags, Galleries) appear on the main navigation bar.
*   **Random Discovery Button:** Toggle the visibility of the floating "Random" action buttons on list pages.
*   **Shake Gesture:** Enable or disable the "Shake to Randomize" feature globally.

## Support & About (`/settings/support`)
*   **App Info:** View the current app version and check for updates.
*   **Diagnostics:** Access developer logs and system diagnostic information.
*   **Resources:** Links to report issues or access official documentation.
