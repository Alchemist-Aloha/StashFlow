# StashFlow Design Specification

## Overview
StashFlow is a mobile-first Flutter application designed to replicate the functionality of the `StashAppAndroid` TV app, optimized for mobile devices with a YouTube-style UI/UX.

## Target Platform
- **Android Only** (initially, with potential for iOS).

## Architecture & Data Flow
The app follows **Clean Architecture**:
- **Presentation Layer**: Riverpod providers and Flutter widgets (YouTube-style UI).
- **Domain Layer**: Entities and Repository Interfaces (business logic, data-agnostic).
- **Data Layer**: Repositories, Data Sources (GraphQL, Local Storage), and Models (DTOs).

## Domain Entities
- **Scenes**: Video content with metadata, markers, and stream URLs.
- **Images**: High-quality images with metadata.
- **Performers**: Individual actors/performers.
- **Studios**: Production houses/studios.
- **Tags**: Categorization and metadata tags.
- **Galleries & Groups**: Collections of scenes and images.

## State Management (Riverpod)
- **AsyncNotifier**: For data fetching (Scenes, Performers, etc.).
- **StateProvider**: For search queries, filters, and UI toggles.
- **Provider**: For repository and data source instances.
- **VideoPlayerProvider**: For managing active video player and mini-player state.

## Navigation (GoRouter)
- **ShellRoute**: For the persistent bottom navigation bar.
- **GoRoute**: For detail screens (SceneDetails, PerformerDetails, etc.).
- **NestedRoutes**: For deep navigation within a section.
- **YouTube-style Player**: A persistent player that can be collapsed into a mini-player or expanded into a full-screen player.

## UI/UX (YouTube-style)
- **Bottom Navigation**: Home, Explore, Subscriptions, Library.
- **Persistent Bottom Player**: A mini-player that stays visible during navigation.
- **Modern Grids**: Video cards with high-quality thumbnails and metadata using Material 3.
- **Top Search Bar**: Accessible from any main tab with filtering options.

## API Integration (GraphQL)
- **Client**: `graphql_flutter`.
- **Operations**: `FindScenes`, `FindPerformers`, `FindStudios`, `FindTags`, `FindGalleries`, `FindGroups`, and `FindImages`.
- **D TO Models**: Autogenerated models for data transfer.
- **Authentication**: Using API key and SSL configuration.

## Video Playback
- **Core Player**: `video_player` with `chewie` or `media_kit`.
- **Scene Markers**: Timeline markers for quick navigation.
- **Playlists**: Multiple scenes with auto-play and shuffle.
- **Stream Quality**: Choice between original, transcoded, and proxy streams.
- **Mini-Player**: Persistent, collapsible player.

## Development Approach
- **Clean Architecture + Riverpod + GoRouter**.
- **Mobile-first UI design**.
- **Direct API mapping from Stash server GraphQL operations**.
